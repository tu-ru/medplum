Create new virtual claim

POST
https://ilm-dev.dha.go.ke/uat-middleware/api/v1
/claims/visit

Creates a new virtual claim for a beneficiary after verifying their consent. Use the OTP strategy when consent was captured via a one-time password, or the biometrics strategy when consent was captured via eKYC or fingerprint — in which case the authorization GUID from POST /api/v1/claims/authorize is passed instead of an OTP.

Create new virtual claim › Request Body

oneOf
Exactly one variant must match.

Decision Table

VariantMatching Criteria
type = object · requires: intervention_codes, patient_id, service_type +1 more
type = object · requires: intervention_codes, patient_id, service_type +1 more
Properties for OTP Visit:
Request payload for starting a visit using OTP-based consent. The patient's one-time password is provided to confirm consent before the claim is created.
intervention_codes
string[] · required

Intervention code(s) for the service scheduled to be offered.

Example: ["SHA-12-001"]
patient_id
string · required

Client Registry identifier of the beneficiary associated with the patient. Example is dummy data.

Example: CR2594484249725-2
service_type
string · enum · required

Type of service being initiated.

Enum values:
CAPITATION
OUTPATIENT
INPATIENT
EMERGENCY
Example: OUTPATIENT
otp
string · required

One-time password sent to the beneficiary's registered contact.

Example: 123456
practitioner_identification_type
string · enum

Optional. Identification type of the attending practitioner (doctor). Supply the three practitioner fields together to attach the doctor's details to the claim.

Enum values:
registration_number
National ID
Alien ID
Refugee ID
Example: registration_number
practitioner_identification_number
string

Optional. Identification number of the attending practitioner (doctor). Provide the identification number, identification type and regulation body together to attach the doctor to the claim if they were not added before. The same doctor details can be supplied through any of the three endpoints — Create Virtual Claim, Add Diagnosis, or Add Billable Line Item. Example is dummy data.

Example: A00000
practitioner_regulation_body
string · enum

Optional. Regulation body the attending practitioner is registered with.

Enum values:
KMPDC
COC
NCK
PPB
Example: KMPDC
