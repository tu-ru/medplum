Create prescription

POST
https://ilm-dev.dha.go.ke/uat-middleware/api/v1
/prescriptions

Creates a new prescription for a patient, capturing the medication items to be prescribed. The prescribing doctor is identified via identification_type (one of registration_number, National ID, Alien ID or Refugee ID) together with regulation_body (one of KMPDC, COC, NCK or PPB).

Create prescription › Headers

X-Facility-Id
string

Facility Registry (FR) code identifying the facility. Required when authenticating with a service-account token.

Example: FID-47-115307-8
X-Facility-Id-Type
string

Type of identifier supplied in X-Facility-Id. Currently only fr-code is supported.

Example: fr-code

Create prescription › Request Body

consent_token
string · required

Patient consent token authorising the prescription request.

Example: K5HLPJVDNV
intervention_code
string · required

SHA intervention code the prescription is issued against.

Example: SHA-18-005
object[] · minItems: 1 · required

List of medication items to be prescribed.

identification_number
string

Identification number of the prescribing practitioner, matching the identification type. Example is dummy data.

Example: A00000
identification_type
string · enum

Type of identifier supplied for the prescribing practitioner.

Enum values:
registration_number
National ID
Alien ID
Refugee ID
Example: registration_number
regulation_body
string · enum

Regulatory body that registered the prescribing practitioner.

Enum values:
KMPDC
COC
NCK
PPB
Example: KMPDC

Create dispense

POST
https://ilm-dev.dha.go.ke/uat-middleware/api/v1
/prescriptions/dispense

Dispenses against an existing prescription, recording the actual products supplied to the patient and the dispensing doctors who fulfilled the prescription.

Create dispense › Headers

X-Facility-Id
string

Facility Registry (FR) code identifying the facility. Required when authenticating with a service-account token.

Example: FID-47-115307-8
X-Facility-Id-Type
string

Type of identifier supplied in X-Facility-Id. Currently only fr-code is supported.

Example: fr-code

Create dispense › Request Body

object[] · required

Products actually dispensed to the patient for this intervention.

consent_token
string · required

Patient consent token authorising the dispense.

Example: K5HLPJVDNV
object[] · required

Doctors associated with the dispense (e.g. prescriber).

intervention_code
string · required

SHA intervention code the dispense is recorded against.

Example: SHA-18-005

Remove a doctor from a prescription

DELETE
https://ilm-dev.dha.go.ke/uat-middleware/api/v1
/prescriptions/doctors

Removes a doctor that was previously associated with a prescription, using the details in the request payload.

Remove a doctor from a prescription › Headers

X-Facility-Id
string

Facility Registry (FR) code identifying the facility. Required when authenticating with a service-account token.

Example: FID-47-115307-8
X-Facility-Id-Type
string

Type of identifier supplied in X-Facility-Id. Currently only fr-code is supported.

Example: fr-code

Remove a doctor from a prescription › Request Body

consent_token
string · required

Consent token authorising the action on the beneficiary's prescription.

Example: K5HLPJVDNV
intervention_code
string · required

SHA intervention code the doctor is associated with.

Example: SHA-18-005
practitioner_registration_number
string · required

Regulatory registration number of the prescribing doctor to remove. Example is dummy data.

Example: A00000
