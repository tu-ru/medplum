Add a protocol to an emergency claim

POST
https://ilm-dev.dha.go.ke/uat-middleware/api/v1
/claims/emergency/protocols

Adds an emergency-care protocol (as a line) to an emergency claim; accepts a JSON body or multipart form-data.

Add a protocol to an emergency claim › Headers

X-Facility-Id
string

Facility Registry (FR) code identifying the facility. Required when authenticating with a service-account token.

Example: FID-47-115307-8
X-Facility-Id-Type
string

Type of identifier supplied in X-Facility-Id. Currently only fr-code is supported.

Example: fr-code

Add a protocol to an emergency claim › Request Body

oneOf
Exactly one variant must match.

Decision Table

VariantMatching Criteria
type = object · requires: consent_token, protocol_code, intervention_code +2 more
type = object · requires: consent_token, protocol_code, intervention_code +2 more
Properties for Protocol Only:
Adds an approved emergency protocol to an emergency claim as a billable line. Sent as a raw JSON body. unit_price must be within the protocol's approved applicableTariff.
consent_token
string · required

Authorization code from emergency claim creation. Signifies an authorised claim.

Example: K5HLPJVDNV
protocol_code
string · required

Code of the approved protocol being billed (from the fetch-protocols response).

Example: PROT-001
intervention_code
string · required

Emergency intervention code the protocol applies to. Emergency interventions belong to the SHA-01 code family.

Example: SHA-01-005
unit_price
string · required

Unit price for the protocol, within the approved applicableTariff. A numeric value sent as a string.

Example: 5000
quantity
string · required

Quantity billed. A numeric value sent as a string.

Example: 1

Creates an EMT (Emergency Medical Transport) claim from a multipart/form-data submission, as part of the emergency-care flow.

Create an EMT claim › Headers

X-Facility-Id
string

Facility Registry (FR) code identifying the facility. Required when authenticating with a service-account token.

Example: FID-47-115307-8
X-Facility-Id-Type
string

Type of identifier supplied in X-Facility-Id. Currently only fr-code is supported.

Example: fr-code
